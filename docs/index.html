üéØ The Million-Step Challenge

Previous LLM systems would inevitably fail after a few hundred steps due to cascading errors. The Towers of Hanoi benchmark with 20 disks requires 1,048,575 steps - a challenge that seemed impossible for AI systems.

1M
First-Ever Achievement: MAKER system successfully solved the 20-disk Towers of Hanoi problem with zero errors across over one million LLM steps
‚àû
Theoretical Scalability: The framework demonstrates logarithmic scaling, theoretically enabling trillions of steps with appropriate reliability guarantees
üí°
Breakthrough Insight: Instead of building bigger LLMs, extreme decomposition + ensemble voting creates reliable systems from smaller, focused models

üî¨ Scientific Breakthrough

The research introduced Massively Decomposed Agentic Processes (MDAPs), a paradigm shift from monolithic AI approaches to ensemble-based reliability:

  • Maximal Agentic Decomposition: Breaking tasks into individual micro-steps
  • First-to-Ahead-by-K Voting: Ensemble decision-making with theoretical guarantees
  • Red-Flagging: Filtering responses that show signs of confusion or errors

üíª Quick Start Example

Get started with just a few lines of code:

# Python MCP client example import asyncio from mcp.client import create_client async def main(): async with create_client("stdio", command="uv", args=["run", "ensample"]) as client: result = await client.call_tool( "ensample.execute_llm_role", { "prompt": "Generate a Python function to calculate fibonacci numbers", "role_name": "CodeGenerator", "voting_k": 3, "ensemble_config": { "models": [ {"provider": "openai", "model": "gpt-4o-mini"}, {"provider": "anthropic", "model": "claude-3-haiku"} ] } } ) print(f"Result: {result.content[0].text}") print(f"Confidence: {result.metadata['confidence_score']}") asyncio.run(main())
